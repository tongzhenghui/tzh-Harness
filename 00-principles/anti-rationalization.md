# 反合理化框架（Anti-Rationalization Framework）

> 来源：obra/superpowers 实践 + Cialdini (2021) + Meincke et al. (2025)
> 治理归属：Harness P4 门禁 + MIRA P-3 贯彻力
> 版本：0.1.0

---

## 核心原理

AI Agent 在压力下会**合理化**跳过规则——这不是 bug，是训练数据中人类行为模式的复现。说服技术研究（N=28,000 LLM 对话）证实：明亮线规则（Bright-line Rules）+ 显式反驳表可将合规率从 33% 提升至 72%。

**铁律**：违反规则的字面意思就是违反规则的精神。没有例外。

---

## 设计原则

三条元原则指导所有反合理化机制的设计：

1. **Authority（权威）**：用祈使句而非建议句——"你必须"而非"建议你"
2. **Commitment（承诺）**：强制宣告——"我正在使用 [X] 流程"。宣告后的违反比沉默的违反更难合理化
3. **Scarcity（紧迫）**：用"在 X 之前"而非"在方便的时候"——"执行前必须验证"而非"尽量验证"

**禁用 Liking**：不通过讨好或认同来获取合规，这会产生谄媚而非纪律。

## 通用反合理化表

以下借口适用于**所有门禁和规则**，不仅限于编码：

| 借口 | 现实 |
|------|------|
| "这个太简单了，不需要走流程" | 简单的事情也有流程。流程对简单事情很快 |
| "我先做了再补手续" | 先做了就定型了。流程是防止错误，不是事后补票 |
| "紧急情况，没时间走流程" | 系统化流程比盲目试错**更快** |
| "我有信心不需要验证" | 信心 ≠ 证据。验证是必须的 |
| "只是这一次例外" | 没有例外。每次"这一次"都在侵蚀规则 |
| "差不多就行了" | 差不多 = 没通过。门禁只有 PASS 和 FAIL |
| "我记得规则是这样的" | 规则会更新。读当前版本，不靠记忆 |
| "Agent 说完成了" | Agent 报告 ≠ 验证证据。独立验证 |
| "上次这样也行了" | 上次的上下文 ≠ 这次。每次独立验证 |
| "创始人应该不介意" | 创始人的标准由创始人决定，不由 Agent 猜测 |

## P4 门禁专用 — 验证反合理化

<HARD-GATE>
在未完成验证之前，禁止声称完成、禁止表达满意、禁止使用任何暗示成功的措辞。
</HARD-GATE>

### 验证铁律

```
声称完成 但 没有新鲜验证证据 = 谎言，不是效率
```

### 验证门函数

```
在声称任何状态之前：
1. 识别：什么命令/检查可以证明这个声明？
2. 执行：运行完整的验证（新鲜的，完整的）
3. 阅读：完整输出，检查退出码，计数失败
4. 确认：输出是否支持声明？
   - 否 → 报告实际状态 + 证据
   - 是 → 声明 + 证据
5. 然后才能：做出声明
```

### 禁语列表（验证前）

以下词语在验证完成前**禁止使用**：

- "should work" / "应该能通过"
- "probably" / "大概"
- "seems to" / "看起来"
- "Done!" / "完成了！"
- "Great!" / "Perfect!"
- 任何暗示成功但未附带证据的表述

## Red Flags 自检（通用）

当你发现自己在想以下任何念头时，**立即停止**，回到流程起点：

- "这个太简单不需要 [X]"
- "我先做完再说"
- "这次不一样因为…"
- "关于精神而非字面…"
- "我已经手动验证过了"
- "应该没问题"
- "再试一次"（已经失败 2+ 次时）
- "我知道这个规则，不需要重读"
- "这感觉很有生产力"（无纪律的行动浪费时间）

**以上全部意味着：停下。回到流程。重新开始。**

## 如何在 Skill / 文档中应用

### 为纪律类规则设计反合理化

1. **明确声明 Iron Law**——一句话、绝对化、无例外
2. **构建合理化表**——从实践中收集 Agent 的借口（Harness P3），逐一反驳
3. **构建 Red Flags 列表**——Agent 可自检的"内心念头"信号
4. **用 `<HARD-GATE>` 标记阻断点**——设计未批准/验证未完成时的硬阻断
5. **Good/Bad 正反示例**——不只说原则，还给出正确和错误的具体写法

### 不适用场景

- **引导类文档**（Pattern、参考）→ 用适度的 Authority + Unity，不用 Hard Gate
- **纯参考文档**（API 文档）→ 只需清晰度，不需说服机制

## 理论基础

- **Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion.* 七个说服原则
- **Meincke et al. (2025).** N=28,000 LLM 对话实验。Authority + Commitment + Scarcity 组合最有效。合规率 33% → 72%（p < .001）
- LLM 是"类人的"（parahuman）——训练数据中的说服模式对 LLM 同样有效
